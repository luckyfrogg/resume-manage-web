## 简历管理端功能
- 简历管理模块
  - 简历列表
  - 新增简历信息
  - 修改简历信息
  - 删除简历信息 

resume表：
  - id:简历id
  - file:简历附件
  - name:姓名
  - gender:性别
  - age:年龄
  - work_year:工作年限
  - location:所在地
  - school_name:毕业院校
  - school_type:学校类型（本科，专科，研究生）
  - graduate_date：毕业时间
  - work_type:工作性质（正式，实习，策软等。。）
  - skills:主要技术栈（下拉多选，没有就新增）
  - evaluation:评估（text类型富文本）
  - create_time:创建日期
  - update_time:修改日期
  - status:简历状态（0：已删除，1：正常）

所有列表支持批量操作


1. 普通管理员只显示被分配的简历列表，超级管理员可以查看所有
2. 新增简历时 选择工作类型时表单支持没有对应工作类型就新增（只有超级管理员可以新增简历）
3. 显示列表时连表查询面试安排的面试状态和面试时间（按更新时间且status==1查最新一条）
4. 新增表单里还需要体现出面试方式，面试时间，指派面试官（超级管理员，管理员A，管理员B）

- 面试安排模块
  - 面试安排列表 
  - 编辑面试安排（弹框修改）
  
  面试安排表：
    - id：面试安排id
    - resume_id:简历id
    - interview_way:面试方式（电话面试，现场面试，视频面试）
    - interview_time:面试时间（具体到时分）
    - interview_status:面试状态（0：初试，1：复试，2：终试，3：通过，4：不通过，5：取消面试）
    - evaluation:评估（text类型富文本）
    - create_time:创建日期
    - update_time:修改日期
    - admin_id:面试官id
    - status:状态（0：已删除，1：正常）
  1. 普通管理员只显示被分配的面试安排，超级管理员可以查看所有
  2. 面试安排列表无新增，初始数据通过新增简历时生成，初始状态为初试
  3. 修改面试安排其实是在面试安排表中新增一条数据不是直接修改面试状态和评估
  4. 当简历删除对应面试安排也删除

- 技术栈表（不一定需要加）
    - id:技术栈id
    - name:技术栈名称

 - 管理员模块
    - 注册管理员
    - 管理员列表
- 管理员表 admin表
    - id：管理员id
    - admin_name:管理员名称
    - login_name:登录名
    - job_number:工号
    - job_email:工作邮箱（后期做发送邮件提醒面试官面试）
    - password:密码（初始密码123456 md5加密）
    - create_time:创建时间
    - update_time:更新时间
    - admin_type:类型（0：超级管理员，1：普通管理员）
    - status:状态（0：禁用，1：正常）
- 登录模块
  - 登录
  - 忘记密码
